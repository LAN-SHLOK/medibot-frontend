## ✨ Premium UI & Logic Features

### 1. Real-Time Biological Avatar Animation
The Medibot Face is entirely driven by live Data.
- **No Pre-rendering:** We avoid static GIFs or generic CSS equalizer bars.
- **Web Audio API Integration:** `script.js` intercepts the live audio stream generated by the backend and processes the frequency domain via an `AnalyserNode`.
- **Dynamic Vector Physics:** 
  - **Eyes:** The `ry` (height) attributes of the overarching SVG eyes compress relative to low-end bass frequencies, allowing the bot to "squint" realistically.
  - **Mouth:** A Quadratic Bezier Curve (`Q`) anchors the center of the mouth line, dynamically pushing downward synchronously with mid-end vocal syllables.

### 2. Multi-Agent "Thought Process" Renderer
While the backend medical council debates a diagnosis, the frontend organically surfaces this hidden mechanic. The `<div id="agentDiscussion">` smoothly renders distinct CSS cards mapping to the active agent (e.g., *Dr. Diagnosis*, *The Skeptic*, *Chief Medical Judge*), allowing users unparalleled transparency into the AI's clinical reasoning.

### 3. Server-Sent Events (SSE) & Voice Abort Switch
- **Parallel Streaming:** Voice Bytecode is streamed over an `EventSource` connection, ensuring total parity between Text and Spoken word with sub-second latency.
- **Hard-Kill Protocol:** Toggling the Voice input off instantly invokes `.pause()`, `.muted = true`, and dynamically purges `.src` attributes. This aggressively mitigates browser buffer-leakage and cleanly silences audio immediately.

### 4. Smart System Handlers
- **Camera WebRTC:** Captures live hardware video feeds directly in the browser, extracting clean base64 matrices to be posted to the Vision Analysis pipeline.
- **Emergency UI Overrides:** Monitors `risk` keys attached to returning payloads. A `HIGH` or `CRITICAL` signal (Tier 3+) universally triggers the `emergency-override` CSS classes, flashing the screen and alarming the user of immediate danger.
- **Anti-Spam Locks:** Suggestive Reinforcement Learning elements instantly assign pointer locks (`btn.disabled = true`) mid-fetch to mitigate accidental asynchronous race conditions.

## 🚀 Setup & Execution

Because the application is built entirely in Vanilla Web technologies, **there is no build step or package manager required.**

1. **Clone & Serve:**
   Simply serve the root directory containing `index.html`. 
   - **VS Code:** Use the *Live Server* extensions.
   - **Python:** Run `python -m http.server 3000` in the directory.
   - **Node:** Run `npx serve .` 

2. **Connect to Backend:**
   Ensure the two external microservices are active before operating the client:
   - Chat Backend running on `http://127.0.0.1:8000`
   - Scanner Backend running on `http://127.0.0.1:8001`
   *(Modify the global endpoint const variables atop `script.js` if running in production/cloud).*
